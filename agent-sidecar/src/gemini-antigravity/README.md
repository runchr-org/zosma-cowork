# Gemini (Google) provider

Adds a one-click **"Sign in with Google"** option (Settings → Authentication →
*Gemini (Google)*) that runs Gemini inference on the signed-in account's own
**Antigravity / Gemini Code Assist** quota. Built entirely in Cowork's layer via
pi's public runtime registries (`registerOAuthProvider` + `registerApiProvider`)
— no edits to the vendored `@earendil-works/pi-*` packages.

First-time sign-in **auto-provisions** the free Code Assist project (via
`onboardUser`, mirroring the official gemini-cli), so the user does not need to
open the Antigravity IDE first.

## The build-time secret — what you must provide

The OAuth code→token exchange requires Antigravity's **client secret**
(`GOCSPX-…`). This is:

- **Antigravity's app credential, not a user credential** — the same value for
  every install (it identifies the *application* to Google, like Antigravity's
  own binary does). Users never see or enter it.
- **Not generated by us** — Google generated it for Antigravity's OAuth client;
  it ships inside the public Antigravity desktop app and is published in
  `github.com/tuxevil/pi-antigravity-rotator` (`src/types.ts` → `CLIENT_SECRET`,
  base64-encoded).
- **Deliberately NOT committed** to this repo: GitHub secret-scanning blocks it
  and a Google client secret shouldn't live in source. `constants.ts` ships a
  placeholder; `scripts/prebuild.mjs` injects the real value into the bundle at
  build time.

**Obtain the value** (it's public, but we don't reproduce it here so it stays
out of this repo): take `CLIENT_SECRET` from `pi-antigravity-rotator`'s
[`src/types.ts`](https://github.com/tuxevil/pi-antigravity-rotator/blob/main/src/types.ts)
(it's base64-encoded there) and decode it — you'll get a `GOCSPX-…` value. It is
the same secret the Antigravity desktop app ships.

### Testers — building from this fork branch

Provide the secret in **one** of these ways before building, then build normally
(`npm install` / `npx tauri build`):

```bash
# Option A — gitignored file (persists across builds)
echo 'GOCSPX-...' > agent-sidecar/antigravity-client-secret

# Option B — env var for a single build
ANTIGRAVITY_CLIENT_SECRET='GOCSPX-...' npx tauri build
```

`prebuild.mjs` bakes it into `index.cjs`. If you skip it, the app still builds
and runs — only *Gemini (Google)* sign-in is disabled, with a clear message.
The OAuth **client ID** (public identifier, not a secret) is already in
`constants.ts`; both are overridable via `ANTIGRAVITY_CLIENT_ID` /
`ANTIGRAVITY_CLIENT_SECRET`.

### Maintainer (upstream) — if this is merged

The secret **cannot be committed** (push protection blocks it). Add
`ANTIGRAVITY_CLIENT_SECRET` as a **GitHub Actions secret** (repo → Settings →
Secrets and variables → Actions) and export it in the build/release workflow so
`prebuild.mjs` injects it. Without it, official builds ship with Gemini (Google)
sign-in disabled (graceful, no breakage). Optional env overrides:
`ANTIGRAVITY_CLIENT_ID`, `ANTIGRAVITY_ENDPOINT`, `ANTIGRAVITY_USER_AGENT`, etc.
(see `constants.ts`).

## Files
- `constants.ts` — client id (+ secret placeholder), scopes, endpoints, model list.
- `oauth.ts` — loopback+PKCE consent, token exchange/refresh, `loadCodeAssist` +
  `onboardUser` project discovery/provisioning.
- `provider.ts` — the `v1internal:streamGenerateContent` translator (pi `Context`
  ↔ Gemini; ported from pi's google provider, transport swapped to Code Assist).
- `index.ts` — registers the OAuth + api providers; injects models on sign-in.
